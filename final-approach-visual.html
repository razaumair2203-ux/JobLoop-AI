System 1: Socratic Engine

What It Actually Is

A structured data collection system with an LLM as the conversational interface. NOT an AI that gets wiser. NOT a chatbot. The intelligence is in the architecture (what to ask, when to stop), not the prompt phrasing.

Architecture Matters Most

Which signal is missing? Ask about it. Has the user already answered? Don't ask again. 3-5 questions max? Stop. This is if/else logic, not AI magic.

Data Accumulation, Not Learning

The engine doesn't get "smarter." The Cloud gets more data. Application #1 needs 5 questions because the Cloud is empty. Application #15 needs 0 because the Cloud is full.

LLM Is the Interface Layer

The LLM makes questions conversational instead of form fields. The phrasing matters, but a decent prompt is 90% of the way there. Nightly optimization adds marginal value.

9 Persona Framings

Same logic, different words. Persona detection is automatic from CV analysis. Research-backed: McKinsey 5 talent pools, Jobvite Job Seeker Quadrant, TopResume 2026 (2K respondents), Huntr 2025 (1.7M apps).

Early Career
Education only or <2yr experience
"Let's find what you DO have." Capstone > internship > coursework > projects. Never make them feel inadequate.
Mid Career
5-15yr experience, established but not senior
Biggest talent pool. Probe for leadership moments, mentoring, cross-team impact. Quantify scope growth over time.
Senior
15yr+ experience, deep domain expertise
Role progression within org. Internal transfers count. Depth = strength, not limitation. Ask about architectural decisions, team building.
Executive
VP/Director/C-level titles
Ask about scope/budget/team, not technical details. Before/after states. Business outcomes.
Career Changer
Different industries across roles
Previous domain = valuable. Probe transferable tools. Don't ask about target domain experience.
Freelancer
Many short engagements, varied clients
Ask for top 3-5 projects, not all 50. Aggregate stats are valid. Client names + portfolio if possible.
Returner
Timeline gap >1 year (voluntary)
Don't ask WHY they took a break. Pre-gap strength + during-gap activity. Domain knowledge doesn't decay.
Laid Off
Recent involuntary separation, urgency signals
Distinct from voluntary returner. Address urgency without panic. Focus on transferable strengths. 9% of job seekers (TopResume 2026), 13% AI-displaced (Huntr 2025).
Military
Military keywords, ranks, service branches
Ask about TYPE not details (classified). Translate rank → leadership. Security clearances = Tier 1 evidence.

What Parser Must NOT Do
  • Never hallucinate numbers — "large team" → has_team_scope: true, NOT team_size: 50
  • Never rewrite user's words — preserve original phrasing
  • Never auto-resolve contradictions — always ask user
  • Never claim verification — we describe, never vouch for truth

Not AI getting smarter. Database getting fuller.

System 2: Outcome Intelligence

What It Actually Is (v3)

A per-user application memory system that enriches the Living Profile Cloud through outcome feedback, enabling smarter CV generation over time. NOT a dashboard showing rejection counts. NOT cross-user learning.

Key Innovation
The Cloud gets smarter about what matters in each niche — for THIS user, from THEIR outcomes. When the user reports "they liked my data pipeline work but wanted more K8s," the system parses this, enriches the Cloud, and silently uses it in future CV generation for similar roles. Zero competitors do this.

System 3: AutoResearch Loop

What It Actually Is

Karpathy's AutoResearch pattern applied to prompt optimization on our synthetic test bank. Not invention — application of known techniques (DSPy, OPRO, TextGrad all exist).

Prior Art
  • DSPy (Stanford, 2023): 10-40% improvement over manual prompting

  • TextGrad: autograd-for-text, LLM feedback as gradients

  • We APPLY known techniques to job search domain. We don't invent.

Summary

Three Systems, Clearly Separated

1. Socratic Engine

What: Structured data collection via conversational questions

Intelligence: Architecture (what to ask, when to stop), not AI magic

Data: Per-user Cloud (private)

Improvement: Cloud fills up (data accumulation), not system learning

Optimization: Hand-crafted templates, manually reviewed

2. Outcome Intelligence (v3)

What: Per-user application memory system that enriches the Cloud

Intelligence: Outcome feedback parsed by DeepSeek Flash → niche-specific signals on Cloud skills

Data: Per-user applications (private, never cross-user)

Improvement: Cloud accumulates niche context → CV gen silently uses it ($0 extra)

Active features: Same company recall, gap accumulation (2+ mentions → Socratic), niche profiles

3. AutoResearch Loop

What: Nightly prompt optimization on our synthetic test bank

Intelligence: Karpathy's keep/discard loop (not invention, application)

Data: Our 45 CV/JD pairs (4 real human + 41 Kaggle-sourced, never user data)

Improvement: Prompts get better → all users benefit equally

Optimization: 2 prompts nightly (cv-gen, jd-parser). 3 manual review.

User data NEVER crosses into AutoResearch.
AutoResearch NEVER uses user outcomes.
Cross-user learning REJECTED after research (Simpson's Paradox, bias amplification).
